## Problem

当前 `main.py` 过长，已经承载了过多职责，包括 AstrBot 命令注册、会话/模式展示、权限确认交互、文件发送入口、ACP 会话管理，以及 `call_opencode` tool 的后台执行编排。随着 slash-command 与 agent tool 两条交互链路继续演进，继续把交互逻辑集中在 `main.py` 会让后续修改变得更脆弱，也会提高理解和测试成本。

## Solution

将 `main.py` 进一步收敛为插件入口与路由层，把命令交互、tool 调用链路、以及与 Coding Agent UX 相关的展示/编排逻辑拆回更清晰的模块边界。优先考虑按职责拆分命令处理、tool 执行入口、以及用户直出与 agent 内部消费两类输出路径，并补充相应测试，避免行为回归。
