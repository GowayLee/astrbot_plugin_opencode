**保留的字段（basic_config 中管理员可见）：**

1. `only_admin` — 仅限管理员使用（bool, default: true）
2. `acp_command` — ACP 启动命令（string, default: "opencode"）
3. `acp_args` — ACP 启动参数（list, default: ["acp"]）
4. `acp_startup_timeout` — ACP 启动超时秒数（int, default: 30）

- `backend_type` — 硬编码为 "acp_opencode"，不需要管理员配置

- `default_agent` — 不再暴露在面板，使用内置默认值 "build"

**1. 在 main.py 的 OpenCodePlugin.**init** 中添加配置迁移函数 \_migrate_config：**

在 `self.config = config` 之后立即调用 `self._migrate_config()`。

`_migrate_config` 方法逻辑：

```python
def _migrate_config(self):
    """将旧配置结构迁移到新结构，确保所有消费端看到完整配置。"""
    basic_cfg = self.config.setdefault("basic_config", {})

    # 1. 迁移 confirm_all_write_ops → allow_file_writes
    if "confirm_all_write_ops" in basic_cfg and "allow_file_writes" not in basic_cfg:
        # 旧字段为 true/false 都意味着"允许写入"，只是确认策略不同
        basic_cfg["allow_file_writes"] = True
    basic_cfg.setdefault("allow_file_writes", True)

    # 2. 填充硬编码默认值（从面板移除但代码仍需要的字段）
    basic_cfg.setdefault("backend_type", "acp_opencode")

    capabilities_default = {"fs_read_text": True, "fs_write_text": True, "terminal": True}
    if "acp_client_capabilities" not in basic_cfg:
        basic_cfg["acp_client_capabilities"] = capabilities_default
    else:
        existing = basic_cfg["acp_client_capabilities"]
        if not isinstance(existing, dict):
            basic_cfg["acp_client_capabilities"] = capabilities_default
        else:
            for key, value in capabilities_default.items():
                existing.setdefault(key, value)

    basic_cfg.setdefault("default_agent", "build")
    basic_cfg.setdefault("default_mode", "ask")
    basic_cfg.setdefault("default_config_options", {})

    if "destructive_keywords" not in basic_cfg:
        basic_cfg["destructive_keywords"] = [
            "删除", "格式化", "清空", "rm\\b", "delete\\b", "format\\b",
            "wipe\\b", "destroy\\b", "shutdown\\b", "reboot\\b", "mkfs",
            "dd\\b", "> /dev/"
        ]

    basic_cfg.setdefault("check_path_safety", False)
    basic_cfg.setdefault("confirm_all_write_ops", basic_cfg.get("allow_file_writes", True))

    # 3. 填充已移除的 tool_config 和 output_config
    tool_cfg = self.config.setdefault("tool_config", {})
    tool_cfg.setdefault("tool_description",
        "在用户电脑上调用 OpenCode 等 AI 智能体 Agent 的工具。当用户有执行编程、处理文档等复杂任务的高级需求时，调用此工具。")
    tool_cfg.setdefault("arg_description",
        "详细的任务描述。保持原意，允许适当编辑以提升精准度，也可以不修改。此参数会被传送给 OpenCode 作为输入。")

    output_cfg = self.config.setdefault("output_config", {})
    output_cfg.setdefault("output_modes", ["ai_summary", "txt_file", "long_image", "full_text"])
    output_cfg.setdefault("max_text_length", 1000)
    output_cfg.setdefault("merge_forward_enabled", False)
    output_cfg.setdefault("smart_trigger_ai_summary", True)
    output_cfg.setdefault("smart_trigger_txt_file", True)
    output_cfg.setdefault("smart_trigger_long_image", True)
```

添加辅助方法：

```python
def _check_and_confirm_destructive(self, event, task, timeout):
    """检查安全策略，如需确认则等待用户回复。返回 (approved, reason)"""
    decision = self.security.evaluate_preflight(task)
    if not decision.requires_confirmation:
        return True, None
    if decision.reason == "file_write_blocked":
        return False, "file_write_blocked"
    # ... 原有确认流程
```

然后在 oc_handler 和 call_opencode_tool 中使用这个方法替代直接调用 is_destructive。
